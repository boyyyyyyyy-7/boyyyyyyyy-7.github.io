## 2. THE FULL CODEBASE
- **Site Version**: ~vS3.9 (Identified via `<script src="background-loader.js?v=S3.9"></script>` in `index.html`).
- **Architecture & Patterns**:
  - **Tech Stack**: Pure vanilla HTML/CSS/JS. No heavy frameworks (no React, Vue, Tailwind).
  - **Aesthetic**: Premium dark space theme utilizing `body::before` and `body::after` radial-gradient animations ("moving stars") across almost all pages.
  - **Game Pages (`[game-name].html`)**: Template-based (approx 80+ files). Each contains a standard header (Back to Games, Fullscreen toggle), an `iframe` pointing to the game source, and a detailed SEO-friendly description section at the bottom (About, Mechanics, Features).
  - **Global UX Fixes**: A global script exists to prevent arrow keys (`ArrowUp`, `ArrowDown`, `ArrowLeft`, `ArrowRight`) and `Space` from scrolling the page when a game is active, ensuring inputs only affect the iframe.
- **Key Files & Folders**:
  - `index.html`: Main landing page. Contains a responsive grid of game cards sorted strategically by "🔥 Hot", "✨ New", and regular games.
  - `background-loader.js`: Critical shared script loaded on every page. Handles dynamic backgrounds and the legacy v2 streak system logic.
  - `ad-manager.js`: Centralized script for handling Google AdSense injections.
  - `sitemap.xml`, `robots.txt`: Managed closely for SEO indexing.
  - `/game`, `/assets`, `/img`, `/js`: Standard asset directories.
  - `/Ant escape`: Source folder for the site's exclusive in-house HTML5 game.
  - `beta-index.html`, `beta-streaks.html`: Development/staging files used for testing new layouts.

## 3. THE STREAK SYSTEM (Current Legacy State)
- **How it works**: Currently baked heavily into `background-loader.js`. It runs on page load, gets the current date, and checks if the user visited today. If not, it logs the visit. It calculates the consecutive day streak by iterating backwards through the stored visit history array, including logic to skip weekends.
- **The "0 🔥" Counter**: Displayed at the top of game pages and in the navigation. It reads the computed streak from `localStorage` and updates DOM elements (`.streak-count` or `#streakNumber`) to show the user's current consecutive daily visits.
- **LocalStorage Keys Touched**:
  - `streakDataV2` (Contains a JSON object with `version`, `visits` array, `freezes`, and `best` streak).
  - `streakFreezes` (Standalone string/integer tracking available streak freezes).
- **Known Bugs (CRITICAL)**: The system is notoriously buggy and needs a full rebuild. Because it relies heavily on client-side time calculations that incorrectly mix UTC and local time concepts, streaks often reset at the wrong time (e.g., 8 PM instead of Midnight). The backwards-iteration calculation is fragile and prone to failure.
